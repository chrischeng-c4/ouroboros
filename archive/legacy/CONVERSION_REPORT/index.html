    Conversion Notes¶

    Important Changes¶

    1. match parameter removed: The match parameter in pytest.raises checked error messages.
    2. In most cases, match patterns were kept in subsequent assertions

    3. If match pattern is critical, add explicit assertion: assert "pattern" in str(exc.value)

    4. await kept in lambda: Async calls are preserved as lambda: await func()

    5. This works with data-bridge-test's expect().to_raise()

    6. Context variables: with pytest.raises(E) as exc_info: → exc = expect(...).to_raise(E)

    7. Variable name changed from exc_info to exc for brevity
    8. Access exception via exc.value (same as pytest)

    Known Limitations¶

    Cannot auto-convert: - Multi-line code blocks (function calls spanning multiple lines) - Async context managers (async with) - Async iterators (async for) - Complex control flow inside with block

    These require manual conversion or should remain as pytest.raises().
